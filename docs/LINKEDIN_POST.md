## 🏗️ **Technical Architecture**

**Built on:**
- OpenClaw runtime
- NVIDIA NIM (FREE tier)
- Alibaba Qwen models
- Whisper (local transcription)
- BM25 search (no vector DB needed)
- Python + Node.js workflows

**Key Design Decisions:**
1. **File-Based Memory** — Survives session restarts, human-readable, version-controllable
2. **YAML Agent Metadata** — Standardized definitions, easy validation, no-code creation
3. **BM25 Search (Not RAG)** — Simple, fast, works well at ~100 articles
4. **Local-First Processing** — Privacy, speed, cost savings, offline capability
5. **Cron + Heartbeat Hybrid** — Flexibility (exact timing vs batch checks)

## 🚀 **Getting Started**

**Clone the repository:**
```bash
git clone https://github.com/azazelpy/friday-ai-blueprint.git
cd friday-ai-blueprint
```

**Setup (30 minutes):**
```bash
# Copy to OpenClaw workspace
cp -r . ~/.openclaw/workspace/

# Bootstrap
cd ~/.openclaw/workspace
./bootstrap.sh

# Configure (edit USER.md, TOOLS.md)
code ~/.openclaw/workspace/USER.md
code ~/.openclaw/workspace/TOOLS.md

# Start using!
openclaw start
```

**Full setup guide:** https://github.com/azazelpy/friday-ai-blueprint/blob/main/docs/SETUP.md

## 💡 **Lessons Learned**

**Building Friday taught me:**

1. **Self-improvement is hard** — Getting AI to improve itself requires safeguards (debate, monitoring, verification, rollback)

2. **Memory is everything** — Stateless agents are like goldfish. Persistent memory transforms the experience.

3. **Documentation writes itself** — With the right system (LLM wiki), documentation becomes a byproduct of operation.

4. **Multi-agent > Single agent** — Specialized agents coordinating beat one generalist every time.

5. **Local-first wins** — Privacy, speed, cost savings. Cloud APIs are overrated for most tasks.

6. **UX matters** — Even the most advanced AI is useless if people can't use it. Examples library was a game-changer.
